We do conventional commits. https://www.conventionalcommits.org/en/v1.0.0/

Current kinds of commits:
- `fix(<scope>): 🐛 a commit that fixes a bug`
- `feat(<scope>): ✨ a commit with new feature`
- `refactor(<scope>): 🔨 a commit that introduces a refactor`
- `style(<scope>): 💄 a commit with cosmetic changes`
- `docs(<scope>): 📚 a commit that improves or adds documentation`
- `wip(<scope>): 🚧: a wip commit`
- `perf(<scope>): ⚡ a commit with performance improvements`
- `revert(<scope>): ⏪ a commit that reverts changes`
- `test(<scope>): 🚨 a commit that adds missing tests or corrects existing tests`
- `chore(<scope>): 🧹 a commit with other changes that don't modify src or test files`
- `build(<scope>): 📦 a commit with changes that affect the build system or external dependencies`
- `ci(<scope>): 🤖 a commit that changes our CI configuration files and scripts`
